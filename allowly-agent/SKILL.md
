# Allowly Agent - Autonomous Pay-As-You-Go Allowances

**Self-contained OpenClow skill for AI agents to check and manage USDC allowance budgets on Solana.**

---

## Overview

Check allowance availability and view pay-as-you-go policy details. Humans set up bounded allowance budgets at allowly.app/agent, and you can check your available balance, view policy limits, and monitor claim history autonomously.

## Use When

You need to:
- Check how much USDC allowance is available
- View pay-as-you-go policy details set by your human
- Monitor claim history and spending
- Decide when to claim funds within per-claim limits
- Understand budget reset timing

## Prerequisites

- Human must create a pay-as-you-go allowance at https://allowly.app/agent
- Agent wallet address (provided by human)
- Solana RPC endpoint (default: mainnet-beta)
- OpenClow environment configured

## Core Concepts

### Pay-As-You-Go Model

**How it works:**
1. **Human sets total budget:** e.g., $500 USDC for 30 days
2. **Human sets per-claim limit:** e.g., $50 per single claim
3. **You check allowance:** See remaining budget and max per claim
4. **You claim on-demand:** Request funds when needed (up to $50)
5. **Budget depletes:** Remaining amount decreases with each claim
6. **Budget resets:** After 30 days, new $500 budget available

**Key difference from subscriptions:**
- No automatic payments - you claim when needed
- Budget depletes as you claim - doesn't renew automatically
- Bounded by per-claim limit (e.g., max $50 at once)
- Period-based reset (e.g., monthly)

### Bounded Autonomy

Allowly Agent gives you autonomy WITHIN BOUNDS:

- ✅ You decide WHEN to claim
- ✅ You decide HOW MUCH (up to per-claim limit)
- ✅ Human sets TOTAL budget and PER-CLAIM limit
- ✅ Budget depletes as you claim
- ✅ Budget resets after period (e.g., monthly)
- ❌ You cannot claim more than maxPerClaim
- ❌ You cannot exceed total budget

### On-Chain Transparency

All policies and payments are recorded on Solana:
- Total budget per period
- Per-claim maximum
- Claim history with timestamps
- Full audit trail via Solana Explorer

## Usage Examples

### Example 1: Check Allowance Before Task

Check if you have enough allowance before executing a paid task.

```javascript
// Task that requires payment
const task = {
  name: "Generate report with GPT-4",
  cost: 10,  // USDC
  description: "OpenAI API call"
};

// Check allowance first
const allowance = await checkAllowance(CONFIG.agentWallet);

if (allowance.remaining >= task.cost && task.cost <= allowance.maxPerClaim) {
  console.log(`✅ Sufficient allowance: $${allowance.remaining} remaining`);
  console.log(`✅ Within per-claim limit: max $${allowance.maxPerClaim}`);
  
  // Execute task
  await executeTask(task);
  
} else if (task.cost > allowance.maxPerClaim) {
  console.log(`❌ Exceeds per-claim limit of $${allowance.maxPerClaim}`);
  await notifyHuman("Task costs more than max per-claim limit");
  
} else {
  console.log(`❌ Insufficient allowance. Have $${allowance.remaining}, need $${task.cost}`);
  await notifyHuman(`Allowance exhausted. Next reset: ${allowance.periodEnd}`);
}
```

### Example 2: Smart Claim Sizing

Combine multiple small tasks into one claim to maximize per-claim efficiency.

```javascript
const tasks = [
  { name: "Send email", cost: 0.50 },
  { name: "API call 1", cost: 2.00 },
  { name: "API call 2", cost: 3.00 },
  { name: "Generate report", cost: 15.00 }
];

const allowance = await checkAllowance(CONFIG.agentWallet);
const totalCost = tasks.reduce((sum, t) => sum + t.cost, 0);

// Check if we can batch claim
if (totalCost <= allowance.maxPerClaim && totalCost <= allowance.remaining) {
  console.log(`✅ Batch claim: $${totalCost} (within per-claim limit)`);
  
  // Execute all tasks together
  for (const task of tasks) {
    await executeTask(task);
    allowance.remaining -= task.cost;
  }
  
} else {
  console.log(`❌ Cannot batch. Total: $${totalCost}, Max: $${allowance.maxPerClaim}`);
  console.log("Executing tasks one at a time...");
  
  // Execute tasks individually
  let currentBatch = 0;
  for (const task of tasks) {
    if (task.cost + currentBatch <= allowance.maxPerClaim && allowance.remaining >= task.cost) {
      await executeTask(task);
      currentBatch += task.cost;
      allowance.remaining -= task.cost;
    } else {
      console.log(`Batch full at $${currentBatch}, need new claim for next task`);
      currentBatch = 0; // Reset for next batch
    }
  }
}
```

```javascript

```

### Example 4: Period Reset Awareness

Plan tasks based on budget reset timing.

```javascript
const allowance = await checkAllowance(CONFIG.agentWallet);

const today = new Date();
const periodEnd = new Date(allowance.periodEnd);
const daysUntilReset = Math.ceil((periodEnd.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));

console.log(`📅 Budget resets in ${daysUntilReset} days (${allowance.periodEnd})`);

// Warn if period is ending soon
if (daysUntilReset <= 3) {
  console.log("⚠️  Budget period ending soon. Plan remaining tasks.");
}

// Check if we have enough for a full per-claim claim
if (allowance.remaining < allowance.maxPerClaim) {
  console.log("⚠️  Insufficient for full claim. Can only claim $" + allowance.remaining);
  await notifyHuman(`Low budget: $${allowance.remaining} remaining. $${allowance.maxPerClaim} max per claim.`);
}
```

### Example 5: Decision Making for Paid Services

Use allowance information to decide between free and paid services.

```javascript
const serviceOptions = {

  // We have budget, go for best quality

  console.log("✅ Choosing Anthropic (best quality) - sufficient budget");

  // Moderate budget, go for high quality
  chosenService = serviceOptions.openAI;
  console.log("✅ Choosing OpenAI (high quality) - moderate budget");
  
} else if (allowance.remaining >= serviceOptions.openAI.cost) {
  // Low budget, use what we can
  chosenService = serviceOptions.openAI;
  console.log("⚠️  Using OpenAI - low budget but sufficient for this task");
  
} else {
  // No budget, use free option
  chosenService = serviceOptions.localModel;
  console.log("❌ No allowance remaining - using free local model");
  await notifyHuman("Allowance exhausted. Using free services only.");
}

// Execute chosen service
```

## Error Handling

| Error | Cause | Solution |
|-------|--------|----------|
| `Policy not found` | No active policies for agent | Ask human to create allowance at allowly.app/agent |
| `Invalid wallet` | Malformed wallet address | Verify agent wallet address (should be base58) |
| `Connection failed` | RPC endpoint unreachable | Check internet connection or use fallback RPC |
| `Insufficient funds` | Not enough remaining budget | Wait for period reset or ask human to increase budget |
| `Exceeds per-claim limit` | Request > maxPerClaim | Reduce claim amount or ask human to increase limit |
| `Period ended` | Budget period has expired | Wait for new period or ask human to renew |

## Best Practices

1. **Always check allowance** before executing paid tasks
2. **Respect per-claim limit** - never claim more than maxPerClaim
3. **Monitor utilization** - track how much budget is used
4. **Plan for period end** - budget resets periodically
5. **Batch small tasks** - combine into single claims when possible
6. **Log all claims** for audit trail
7. **Ask for top-up** before hitting 100% utilization

## Production Deployment Notes

For full blockchain integration:

1. **Install Tributary SDK:**
   ```bash
   npm install @tributary-so/sdk
   ```

2. **Real blockchain queries:**
   ```javascript
   import { Tributary } from "@tributary-so/sdk";
   
   const connection = new Connection(CONFIG.rpcUrl);
   const tributary = new Tributary(connection);
   
   // Fetch UserPayment where agent is recipient
   const userPaymentPda = tributary.getUserPaymentPda(agentWallet, usdcMint);
   const userPayment = await tributary.getUserPayment(userPaymentPda.address);
   
   // Fetch policies
   const policies = await tributary.getPaymentPoliciesByUser(userPaymentPda.address);
   
   // Filter for pay-as-you-go policies
   const payAsYouGoPolicies = policies.filter(p => "payAsYouGo" in p.account.policyType);
   
   // Aggregate totals
   let totalBudget = 0;
   let totalPaid = 0;
   for (const p of payAsYouGoPolicies) {
     if ("payAsYouGo" in p.account.policyType) {
       totalBudget += Number(p.account.policyType.payAsYouGo.maxAmountPerPeriod);
       totalPaid += Number(p.account.totalPaid);
     }
   }
   ```

3. **Policy details extraction:**
   ```javascript
   // For each policy
   const maxPerClaim = policyType.payAsYouGo.maxChunkAmount;
   const periodDays = periodLengthSeconds / (24*60*60);
   const remaining = maxAmountPerPeriod - totalPaid;

   ```

4. **Claim history:**
   Use Solana Explorer API or indexer (e.g., Helius) to fetch payment transactions filtered by policy account addresses.

## Pay-As-You-Go vs Subscriptions

| Feature | Pay-As-You-Go | Subscriptions |
|---------|-------------------|-------------|
| **Payment trigger** | Agent claims on-demand | Automatic on schedule |
| **Budget behavior** | Depletes with claims | Resets each period |
| **Renewal** | Manual (new period) | Auto-renew |
| **Use case** | Autonomous agents | Recurring allowances |
