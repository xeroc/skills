# LibHunt Platform Guide

## Platform Overview

**LibHunt** is an open-source project discovery and recommendation platform designed to help developers find the best libraries and tools for their projects. Created by Stan Bright (@StanBright), LibHunt operates as a "go-to website for developers looking for libraries and open source projects to get their work done."

- **URL**: https://www.libhunt.com
- **Creator**: Stan Bright (Software Engineer with 15+ years experience)
- **Category**: Developer Tools / Open Source Discovery Platform
- **Business Model**: Sponsorships and partnerships
- **License**: Lists are licensed under CC BY-SA 4.0

**How it works**: LibHunt monitors everything posted on Reddit, HackerNews & Dev.to in almost real-time. When an open-source repository gets mentioned, they record it and use this data to compile lists of popular and similar projects.

## Who Should Use This

LibHunt is ideal for:

### ✅ Perfect Fit:
- **Open-source libraries and frameworks** in popular programming languages
- **Developer tools and utilities** that solve specific technical problems
- **Projects with strong GitHub presence** (stars, contributors, activity)
- **Libraries mentioned organically** on Reddit, HN, or Dev.to
- **Cross-platform tools** that work across multiple environments
- **Projects with clear documentation** and examples

### ❌ Poor Fit:
- Proprietary software or closed-source projects
- Mobile apps or consumer-facing products
- Projects without GitHub repositories
- Brand new projects with no social mentions
- Non-technical products or services
- Projects that aren't developer-focused

## Submission Process

**Important**: LibHunt is primarily an **automated discovery platform**, not a manual submission site. However, there are ways to increase your visibility:

### Step 1: Get Organic Mentions
1. **Build a quality open-source project** on GitHub
2. **Share your project organically** on:
   - Reddit (r/programming, r/opensource, language-specific subreddits)
   - Hacker News (Show HN posts)
   - Dev.to articles and discussions
3. **Wait for automatic discovery** (LibHunt monitors these platforms)

### Step 2: Contact for Manual Inclusion
1. **Email the founder** at stan@libhunt.com
2. **Subject line**: "Project suggestion for LibHunt: [Your Project Name]"
3. **Include**:
   - GitHub repository URL
   - Brief description of what it does
   - Programming language/category
   - Why it would be valuable to the LibHunt community

### Step 3: Suggest Alternatives
1. **Find similar projects** on LibHunt using the URL replacement trick:
   - Change `github.com/user/repo` to `libhunt.com/user/repo`
2. **Click "Suggest an alternative"** on relevant project pages
3. **Submit your project** as an alternative to similar tools

### Step 4: Community Engagement
1. **Engage with the Bluesky community** (@StanBright)
2. **Contribute to discussions** about similar projects
3. **Be helpful** in developer communities where your project fits

## Requirements

Since LibHunt is automated, focus on these project quality indicators:

### GitHub Repository Must Have:
- **Clear README** with project description
- **Good documentation** explaining usage
- **Active development** (recent commits)
- **Proper licensing** (open source license)
- **Stars and engagement** from real users

### Social Presence Requirements:
- **Legitimate mentions** on Reddit/HN/Dev.to
- **Real community engagement** (not spam or self-promotion)
- **Value-driven discussions** about your project's benefits

### Project Quality Indicators:
- **Solves a real problem** that developers face
- **Well-maintained codebase** with regular updates
- **Responsive maintainers** who engage with issues/PRs
- **Clear use cases** and examples

## Strategy & Tips

### Best Practices:
1. **Focus on organic growth** first - build something genuinely useful
2. **Engage authentically** in developer communities before promoting
3. **Create valuable content** around your project (tutorials, comparisons)
4. **Time your social shares** when relevant discussions are happening
5. **Build relationships** with other open source maintainers

### Optimal Timing:
- **Tuesday-Thursday** for Reddit/HN posts (higher engagement)
- **Morning hours** in US timezones (9-11 AM EST)
- **When relevant discussions** are already happening
- **After major releases** or significant updates

### Maximize Visibility:
1. **Use the alternatives feature** - suggest your project on similar tools' pages
2. **Target language-specific sites** (js.libhunt.com, python.libhunt.com, etc.)
3. **Create comparison content** that naturally mentions your project
4. **Participate in "awesome lists"** in your programming language
5. **Cross-reference** with other discovery platforms (AlternativeTo, etc.)

## Common Mistakes / Gotchas

### What Causes Poor Performance:
1. **Excessive self-promotion** without community value
2. **Poor project documentation** or unclear use cases
3. **Inactive repositories** with stale code or unresponsive maintainers
4. **Spam-like behavior** across multiple platforms
5. **Targeting wrong communities** (consumer apps in developer spaces)

### Technical Issues:
- **LibHunt sometimes has 500 errors** - Don't assume your project was rejected
- **Alternative suggestions can be inaccurate** - The algorithm isn't perfect
- **Category placement may be wrong** - Contact stan@libhunt.com for corrections
- **Language detection issues** - Make sure your repo's primary language is correctly tagged

### Platform Limitations:
- **No direct submission form** - Must rely on social mentions or email contact
- **Automated categorization** can miscategorize projects
- **No guaranteed inclusion** even with organic mentions
- **Limited feedback** on why projects aren't included
