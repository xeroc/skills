# TrustMRR Platform Guide

## 1. Platform Overview

**TrustMRR** is a verified startup revenue database and acquisition marketplace created by Marc Lou, a well-known indie hacker who also built DataFast, ShipFast, and other popular tools. The platform solves the problem of fake revenue screenshots in the startup community by requiring direct API connections to payment processors for verification.

- **URL**: [trustmrr.com](https://trustmrr.com)
- **Category**: Startup Directory / Revenue Database / Acquisition Marketplace
- **Launched**: Late 2025 (November)
- **Creator**: Marc Lou (@marclou)
- **Primary Purpose**: Showcase verified startup revenues and facilitate acquisitions

## 3. Who Should Use This

TrustMRR works best for:

- **SaaS startups** with recurring revenue models
- **Indie hackers** building in public
- **Solopreneurs** targeting other entrepreneurs
- **B2B tools** serving the maker community
- **Companies with Stripe/payment processor data** (verification required)
- **Startups seeking acquisition** or co-founders
- **Revenue-generating startups** ($100+ monthly preferred)

**Not ideal for**:
- Pre-revenue startups
- Companies without payment processor data
- Easily cloned products (sharing revenue may attract copycats)
- Enterprise-only B2B products (audience mismatch)

## 4. Submission Process

### Step 1: Prepare Your Payment Provider
- Must have Stripe, LemonSqueezy, Polar, DodoPayment, or RevenueCat
- Generate read-only API key from your payment processor
- Ensure you have verified revenue data (preferably $100+/month)

### Step 2: Submit Your Startup
1. Visit [trustmrr.com](https://trustmrr.com)
2. Click "Add startup" button
3. Connect payment provider with read-only API key
4. Revenue gets automatically verified and profile created
5. Complete additional details in dashboard

### Step 3: Complete Your Profile
- Access dashboard at [trustmrr.com/dashboard](https://trustmrr.com/dashboard)
- Add startup name, description, logo, website URL
- Set category and tech stack
- Add founder X handle for founder page inclusion
- Upload screenshots and product details

## 8. Common Mistakes / Gotchas

### Technical Issues:
- **API key expiration**: Revenue stops updating if API keys expire (update in dashboard)
- **Metric discrepancies**: TrustMRR revenue may differ from provider dashboard by up to 30% due to different calculation methods
- **Multiple products**: Need separate payment accounts for each product listing
- **Provider limitations**: Not all payment processors supported (check compatibility first)

### Strategic Mistakes:
- **Incomplete profiles**: Missing descriptions hurt discoverability and credibility
- **Ignoring community**: Passive listings get less engagement than active participants
- **Wrong timing**: Listing before having meaningful revenue ($100+/month threshold)
- **Category mismatch**: Choosing wrong category reduces targeted discovery

### Marketplace Pitfalls:
- **Unrealistic pricing**: Overvalued startups sit without offers
- **Poor preparation**: Incomplete due diligence materials slow deal process
- **Anonymous confusion**: Stealth mode reduces buyer confidence
- **Escrow complications**: Missing asset details can cause Escrow.com rejections
